Panoptic Bridge Management

Image Copyright ©Waldeck


The project is a digital examination environment and asset management system, where engineers can examine an asset (typically a bridge) using the website we created and the tools setup. It largely involves the use of the "Virtual Tour": a pointcloud-based viewer that uses potree, an open-source WebGL-based point cloud renderer, and has an image and a BIM viewer embedded within, alongside some AI elements for aiding identification of defects upon the asset. This project is our entire team's effort, so I'll just focus on what I worked on.

Over the three years with the team, I adapted the Autodesk Forge Viewer, which is what is used on the Virtual Tour for displaying BIM models. My work beyond this focused on the front-end development, ensuring the datasets are up-to-date with suppliers, creating and maintaining the automated Python background tasks, and testing both the website (including the API connections to the database) and the Virtual Tour. This involved the use of Jira for bug tracking and sometimes Postman for the API-related bugs. I was also in charge of the tutorials, user and workflow documentation for the latest versions of the system.

This project is near and dear, as they say, and I'm very proud of what we managed to make. Check out the most recent info on it - here!

Artie

Artie is a personal project of mine that I've been working on since September 2021! He's a voice assistant using AI and Machine Learning, built in Python. Currently, his functionality involves a login system, use of the wikipedia API, telling the user a joke, and locking or shutting down the computer.
In many ways, I want Artie to be a program that allows for hands-free use of a PC, which can be very beneficial for people with disabilities that cause muscular atrophy - or even just for users who are busy.

In the future, his functionality will also include: more permanent memories (a brain file is in progress!), getting the weather for a given location, connections with spotipy and the ability to open certain applications (e.g. Word, Excel).
My biggest aim for the current version is to package him into an executable file, without the need to install Python or any dependencies, so he's a bit more user-friendly. Once this is setup, additional functionality will be much easier to implement.
Out of all of my projects, Artie is the most long-term and is undergoing a 2025 overall, as the original repo is very outdated for a AI/ML project.

FYP - AI: Automatic News Extraction

This project, officially titled "Artificial Intelligence: Automatic News Extraction" was for my final year project - aka my dissertation - and is open-source. My project uses a Selenium webscraper to get news articles from the BBC News website, based on a users' query. This data is then analysed and graphed using hot topic extraction.
For future development, I would have liked to increase the corpus of the project, by adding more websites to the webscraper and implementing further search features, along with a machine learning algorithm that ignores potentially "fake" or malicious news.

However, there were issues when I researched this further, as BBC and many other news sites have strict rules and regulations regarding "bots" and their usage on their sites, so this project remains an academic proof-of-concept.

Machine Learning Coursework

For my machine learning module in second year, I used Stack Overflows' developers survey to predict a developers salary, based on the features selected. This project is unique in that it is based and runs on Kaggle, using Jupyter Notebook, which allowed me to run the CPU-intense algorithms. It is one of my favourite projects, as the data had some fascinating results.
In the end, I selected 13 features, including "Gender", "Sexuality" and "Trans" (a yes/no question), as these were the ones I was most curious about - were these factors impactful on a developers' salary? Ultimately, my model was rather successful (around 70%) and utilised clustering for prediction.

If I were to return to this project, I'd love to repeat this with more samples (Stack Overflow repeat this survey annually) and see how this may differ over the years - and if the increasing use of AI has impacted these results.
