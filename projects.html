Artie

Artie is a personal project of mine that I've been working on since September 2021! He's a voice assistant in the making, built in Python.
Currently, his functionality involves a login system, use of the wikipedia API, telling the user a joke, and locking or shutting down the computer.
In the future, his functionality will also include: getting the weather for a given location. In a more vague sense, I will be implementing some machine learning so that Artie can retain "memories" in a more permanent manner, as well as giving him a much wider functionality that allows him to be used casually.
Out of all of my projects, Artie is the most long-term and will be continuously updated over the years.

Final Year Project - BBC Article Webscraper

This project, officially titled "Artificial Intelligence: Automatic News Extraction" is for my final year project - aka my dissertation - and is open-source.
Essentially, my project uses a Selenium webscraper to get news articles from the BBC News website, based on a users' query. This data is then analysed and graphed using hot topic extraction.
For future development, I would like to increase the corpus of the project, by adding more websites to the webscraper and implementing further search features, along with a machine learning algorithm that ignores potentially "fake" or malicious news.

Service-Centric and Cloud Computing (SCC)

This application was created for my SCC module in third year. It uses the Springboot framework in Java to host an Apache Tomcat 9.0 server. Using this server and a rest controller, I had several function calls: /weather, /user, /query, /submit and /intent.
This coursework was a particularly interesting challenge as, at the time, I didn't have much experience with Java and had never used and barely even heard of REST services. Although my final application had some issues (a lack of non-volatile data storage for example), I'm very proud of this one.


Machine Learning

For my ML module in second year, I used Stack Overflows' developers survey to predict a developers salary, based on the features I selected. This project is unique in that it is based and runs on Kaggle, using Jupyter Notebook, which allowed me to run the CPU-intense algorithms.
In the end, I selected 13 features, including "Gender", "Sexuality" and "Trans" (a yes/no question), as these were the most interesting ones. Ultimately, my model was rather successful (around 70%) and utilised clustering for prediction.


Practical Project Management

This module in 2020 was interesting for many reasons. Originally, this was a group-work with five people. However, due to circumstances beyond our control, our group quickly dwindled in numbers to just two - myself and Aurora.
To ensure that we were able to complete this in time, we split the project in half: I created the programs' code and Aurora dealt with the design and documentation, which worked out well!
The final product was MoveBuddy, an application built in Qt with a GUI, that displays the crime statistics for a given area in Nottingham. This was intended as a prototype and, in future versions, could be extended in location and features. For example, users could create accounts and login, registering their home location and getting relevant local information, as well as a chat system. We felt this would be a very useful application, especially for students, who may not know what an area is like before they move for University.
